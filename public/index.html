Study with live controls

The session panel now owns order, reviewed visibility, and difficulty filters at runtime, so you can tighten or loosen the deck without touching cards.md.

  • Open Show session settings for live order, reviewed-visibility, and multi-select difficulty filters.
  • Use Skip (0) as the “park this card for later” difficulty level.
  • Difficulty and review changes save immediately back to cards.md, while the live filters react instantly in the UI.

How to use it

The top banner keeps the live context visible while the session settings stay one toggle away.

  • Use Start studying to enter the compact reader-first layout.
  • The top banner keeps four quick facts together: position, last reviewed, on-card time, and total session time.
  • Reveal answer shows the back, and Hide answer conceals it again.
  • Use ← / → to move, 0–5 to rate difficulty, Space / Enter to reveal or hide, and R to toggle reviewed status.
  • When Hide reviewed is active, reviewed cards disappear from the cycle immediately after you mark them.

cards.md stays focused

The file now stores card content and per-card metadata only; session settings live in the UI.

  • Each card still keeps app-managed metadata such as id, difficulty, and last_reviewed.
  • difficulty: 0 means the card is skipped until you include Skip (0) in the live filter again.
  • Markdown, Mermaid diagrams, and KaTeX math render directly inside the card faces.
  • Legacy file-level settings are no longer needed; the app now derives the active session entirely from the runtime controls.

No cards in this live view

The current runtime settings hide every card. Re-enable one or more difficulty levels, show reviewed cards again, or move skipped cards back above 0.
